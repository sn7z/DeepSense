# 🧠 DeepSense — Explainable AI Multimedia Deepfake Detection

> A forensic-grade deepfake detection platform for images, audio, and video — powered by deep learning and explainable AI.

---

## About

DeepSense is an end-to-end deepfake detection system that analyses multimedia content across three modalities: **images**, **audio**, and **video**. Beyond a simple real/fake prediction, DeepSense provides **explainability** — showing *why* the model reached its verdict through Grad-CAM heatmaps, spectral feature graphs, and natural-language explanations generated by a **Gemini LLM**.

Built for researchers, journalists, and security analysts who need not just a verdict, but evidence.

---

## Overview

| Modality | Model | Explainability |
|----------|-------|----------------|
| 🖼️ Image | XceptionNet | Grad-CAM heatmap |
| 🎵 Audio | CNN + BiLSTM | Waveform, MFCC, Spectral Centroid, ZCR |
| 🎥 Video | XceptionNet + LSTM | Grad-CAM on key frames |

All results are passed to a **Gemini LLM** which synthesises the prediction, confidence score, and visual evidence into a plain-language forensic explanation.

## Deep Learning Models

### 🖼️ Image Deepfake Detection

**Model:** XceptionNet

**Process:**
1. Image preprocessing and normalisation (resize to 299×299, pixel scaling)
2. Feature extraction using XceptionNet's depthwise separable convolutions
3. Binary classification via sigmoid output — **Real** or **Fake**

**Explainability:**
- **Grad-CAM** heatmaps highlight the facial regions (eyes, mouth, skin boundaries) that most influenced the prediction, overlaid on the original image.

---

### 🎵 Audio Deepfake Detection

**Model:** CNN + BiLSTM

**Audio Processing Pipeline:**
1. Resample audio to **16 kHz**
2. Extract **MFCC features** (40 coefficients, 500 time frames)
3. Pad or truncate to a fixed-length feature matrix
4. Feed into model as a 2D feature array

**Explainability Graphs:**
| Graph | What It Reveals |
|-------|----------------|
| Waveform | Temporal signal behaviour — abrupt changes or suspiciously uniform regions |
| MFCC Heatmap | Coefficient patterns irregular in synthetic speech |
| Spectral Centroid | Frequency instability characteristic of AI-generated voice |
| Zero-Crossing Rate | Abnormal signal irregularity common in cloned audio |

These graphs are rendered interactively and also passed as images to the Gemini LLM for evidence-based explanation.

---

### 🎥 Video Deepfake Detection

**Method:**
1. **MTCNN** face detection on sampled frames
2. Face regions resized to 299×299 and preprocessed for Xception
3. Temporal modelling via **LSTM** across 30 extracted frames
4. Softmax classification — **Real** or **Fake**

**Explainability:**
- **Grad-CAM** applied to the Xception backbone (accessed via the `TimeDistributed` wrapper) on up to 6 key frames
- Visualises temporal face inconsistencies across the clip

## Explainable AI (XAI)

### Image & Video
Grad-CAM (Gradient-weighted Class Activation Mapping) generates heatmaps by computing gradients of the predicted class with respect to the last convolutional layer's activations. Hot regions indicate areas the model found suspicious.

**Typical artifacts detected:**
- Face boundary inconsistencies
- Unnatural skin blending
- Eye and mouth distortions
- Texture discontinuities around the hairline

### Audio
Spectral analysis detects synthetic voice artifacts through:
- **MFCC heatmap irregularities** — unnatural coefficient distributions
- **Unstable spectral centroid** — erratic frequency centre-of-mass over time
- **Abnormal zero-crossing rate** — signal noise inconsistent with natural speech

### AI Forensic Explanation (Gemini LLM)
After detection and XAI generation, the system sends the following to the **Gemini LLM**:
- Prediction label (REAL / FAKE)
- Confidence score
- Explainability images (Grad-CAM overlays / spectral graphs)

Gemini returns a plain-language forensic summary explaining *why* the media is flagged, citing visual evidence from the uploaded images.

## Setup & Run

### Prerequisites

- Python 3.9+
- A **Google Gemini API key** (set as environment variable)
- Pretrained model files placed in the `models/` directory (see structure above)

---

### 1. Clone the Repository

```bash
git clone https://github.com/your-org/deepsense.git
cd deepsense
```

### 2. Create a Virtual Environment

```bash
python -m venv venv
source venv/bin/activate        # Linux / macOS
venv\Scripts\activate           # Windows
```

### 3. Install Dependencies

```bash
pip install -r requirements.txt
```

### 4. Set Environment Variables

```bash
export GEMINI_API_KEY="your_gemini_api_key_here"   # Linux / macOS
set GEMINI_API_KEY=your_gemini_api_key_here         # Windows
```

### 5. Add Pretrained Models

Place your pretrained model files in the correct paths:

```
models/image_models/xception_deepfake_model.keras
models/audio_models/my_model.h5
models/video_models/XceptionNet.keras
```

### 6. Run the App
## Double Click the "run.bat" file to start the app, or run:

```bash
streamlit run app.py
```

The app will open at `http://localhost:8501`.

## How It Works — End to End

1. **Upload** a media file via the Streamlit interface
2. The file is **preprocessed** and passed to the relevant deep learning model
3. A **prediction** (REAL / FAKE) and confidence score are returned
4. **Explainability** is generated — Grad-CAM overlays for image/video, spectral graphs for audio
5. The prediction, confidence, and visual evidence are sent to **Gemini LLM**
6. A plain-language **forensic explanation** is displayed alongside the results
